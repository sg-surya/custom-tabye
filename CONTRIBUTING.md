# Contributing to Minimal Tab

Thank you for your interest in contributing to Minimal Tab! This guide will help you get started.

---

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in [Issues](https://github.com/sg-surya/custom-tabye/issues)
2. If not, create a new issue with:
   - Clear title describing the issue
   - Steps to reproduce the bug
   - Expected vs actual behavior
   - Screenshots if applicable
   - Chrome version and OS

### Suggesting Features

1. Check existing issues for similar suggestions
2. Create a new issue with:
   - Clear title with "Feature Request" prefix
   - Detailed description of the feature
   - Why this feature would be useful
   - Any mockups or examples

### Pull Requests

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/custom-tabye.git
   ```
3. **Create** a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/bug-description
   ```
4. **Make** your changes
5. **Commit** with clear messages:
   ```bash
   git commit -m "Add: descriptive commit message"
   ```
6. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open** a Pull Request

---

## Commit Message Guidelines

Use clear, descriptive commit messages:

- `Add: [feature name]` - New features
- `Fix: [bug description]` - Bug fixes
- `Update: [change description]` - Updates to existing features
- `Docs: [description]` - Documentation changes
- `Refactor: [description]` - Code refactoring

---

## Code Style

- Use consistent indentation (2 spaces)
- Keep code clean and readable
- Add comments for complex logic
- Test your changes before submitting

---

## Getting Help

- GitHub Discussions: For questions and general discussion
- GitHub Issues: For bug reports and feature requests
